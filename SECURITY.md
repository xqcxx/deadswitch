# Security Policy

## Supported Versions

We release patches for security vulnerabilities. Which versions are eligible for security updates:

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within DeadSwitch, please send an email to security@deadswitch.io. All security vulnerabilities will be promptly addressed.

Please include the following information in your report:

- Type of vulnerability
- Full paths of source file(s) related to the manifestation of the vulnerability
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Response Timeline

- Initial response: Within 48 hours
- Assessment complete: Within 5 business days
- Fix released: Depends on severity (typically 30 days or less)

## Smart Contract Security

Given that DeadSwitch handles valuable assets on the blockchain, we take smart contract security very seriously:

- All contract changes undergo thorough review
- Critical paths have extensive test coverage
- We encourage community security audits
- Bug bounty program details will be announced soon

## Disclosure Policy

When we receive a security bug report, we will:

1. Confirm receipt of the vulnerability report
2. Work to verify the vulnerability
3. Develop a fix and test it
4. Release the fix and publicly disclose the issue
